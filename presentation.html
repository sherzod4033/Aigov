Решение

Что такое Open-NotebookLM?

🤖

Локальный AI-ассистент для документов

Работает полностью локально — не нужен интернет и внешние API. Генерация текста и векторные эмбеддинги запускаются через Ollama на вашем сервере.

Загрузите документы → система их индексирует → задавайте вопросы → получайте точные ответы со ссылками на источники.

✂️ Шаг 1 — Chunking

  • Документ разбивается на фрагменты (чанки) по ~450 токенов
  • Структура документа учитывается при разбиении
  • Каждый чанк сохраняет метаданные: источник, раздел, страница
  • Поддержка PDF, DOCX, TXT и сканов через OCR

🔍 Шаг 2 — Retrieval

  • Вопрос превращается в вектор (embedding)
  • Параллельно идёт векторный и лексический (BM25) поиск
  • Результаты объединяются через RRF
  • Follow-up вопросы конденсируются в самостоятельный запрос

🧠 Шаг 3 — Generation

  • Найденные чанки передаются в контекст LLM
  • Gemma 3n генерирует ответ на языке вопроса (ru / tj)
  • Domain-profile настраивает стиль промпта
  • Ответ содержит цитаты и ссылки на источники

Гибридный поиск

Vector + BM25 → RRF

🧮 Векторный поиск

Находит фрагменты по смыслу. Работает даже если слова в вопросе и документе разные.

nomic-embed-text · cosine similarity

🔤 BM25 поиск

Точный поиск по словам. Незаменим для статей закона, числовых норм и терминов.

Token frequency · Exact match
⇓ ⇓

RRF

Reciprocal
Rank Fusion

⇓
Топ-K чанков

📐 Формула RRF

Оба результата объединяются по рангу, а не по сырому score:

score =
  1/(k + vec_rank)
+ 1/(k + bm25_rank)

// k = 60

📍 Метаданные чанков

Каждый чанк хранит путь в структуре документа:

source: "НК РТ 2024.pdf"
section: "СТАТЬЯ 12"
page: 47

Шаг 3 · Готовые чанки с метаданными

  • Полный RAG пайплайн с гибридным поиском (Vector + BM25 + RRF)

  • Multi-notebook рабочие пространства
  • Domain Profiles: general, tax, legal
  • Docker Compose — одной командой
