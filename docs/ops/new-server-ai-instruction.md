# Инструкция Для Другого ИИ: Полная Настройка Нового Сервера

Ниже готовый промпт для другого ИИ. Его можно вставить как есть и только заменить значения в угловых скобках.

```text
Твоя задача: полностью развернуть мой проект на новом Ubuntu-сервере с публичным доступом через Nginx, установить Ollama, скачать нужные модели, запустить всё и проверить, что сайт доступен извне.

Работай до полного результата, не останавливайся на полпути. Если находишь проблему, сам диагностируй и исправляй. Если доступ по SSH не работает, сразу сообщи, что именно не так.

Входные данные:
- IP сервера: <SERVER_IP>
- SSH логин: ubuntu
- SSH private key: <ABSOLUTE_PATH_TO_PEM>
- GitHub repository: <REPO_URL>
- Имя проекта на сервере: <PROJECT_NAME>
- Нужные Ollama модели:
  - <MODEL_1>
  - <MODEL_2>
  - <MODEL_3>
- Если нужно создать admin:
  - username: <ADMIN_USERNAME>
  - password: <ADMIN_PASSWORD>

Что нужно сделать:
1. Подключиться к серверу по SSH.
2. Проверить базовое состояние сервера:
   - hostname
   - память
   - диск
   - наличие GPU
   - наличие docker/nginx
3. Установить всё необходимое:
   - git
   - curl
   - docker
   - docker compose
   - nginx
   - ollama
4. Склонировать репозиторий на сервер, например в `/home/ubuntu/<PROJECT_NAME>`.
5. Изучить структуру проекта и определить реальный способ запуска:
   - docker compose
   - docker
   - npm
   - python
   - systemd
   Не угадывай, сначала посмотри README, docker-compose.yml, package.json, requirements.txt, Dockerfile, .env.example и похожие файлы.
6. Настроить проект для production:
   - создать `.env`, если он нужен
   - прописать корректные переменные окружения
   - если backend в Docker должен обращаться к Ollama на хосте, обеспечь рабочий доступ
7. Установить и запустить Ollama.
8. Обязательно скачать все указанные модели через `ollama pull`.
9. Если backend запущен в Docker, проверь доступ к Ollama из контейнера.
10. Если Ollama слушает только `127.0.0.1` и контейнер не может достучаться, исправь это:
   - настрой `OLLAMA_HOST=0.0.0.0:11434`
   - сделай override для systemd
   - перезапусти `ollama`
   - проверь, что порт `11434` реально слушается не только на localhost
11. Поднять приложение.
12. Если проекту нужна инициализация БД, выполни её.
13. Если я попросил создать администратора:
   - найди штатный способ
   - если штатного способа нет, создай через код или БД аккуратно
   - проверь логин
14. Настроить хостовый Nginx так, чтобы любой мог открыть сайт по `http://<SERVER_IP>`:
   - проксируй frontend или backend в зависимости от архитектуры проекта
   - если frontend в контейнере на `127.0.0.1:3000`, проксируй туда
   - если API должен быть доступен через тот же домен, сохрани рабочую маршрутизацию
15. Добавить автозапуск после перезагрузки:
   - либо через systemd unit для docker compose
   - либо другим корректным способом
16. Проверить всё снаружи:
   - сайт открывается по IP
   - health endpoint отвечает
   - Ollama доступен для backend
   - список моделей реально виден приложению, если проект это поддерживает
   - логин admin работает, если был создан

Отдельные обязательные правила:
- Ничего не удаляй без необходимости.
- Не используй destructive-команды.
- Не меняй git remote и не пушь.
- Если сервис не работает, читай логи и исправляй причину, а не просто перезапускай бесконечно.
- Если в проекте frontend ожидает один API route, а backend отдаёт другой, найди и исправь несовпадение минимальным способом.
- Если есть проблема именно в сетевой связности Docker -> host Ollama, исправляй сетевую конфигурацию, а не маскируй ошибку.
- После каждого важного шага делай короткую проверку.
- В конце дай только фактический отчёт: что установлено, что запущено, какой URL, какие логины созданы, какие модели скачаны, какие сервисы активны.

Минимальный чек-лист проверки в конце:
- `curl http://<SERVER_IP>`
- `curl http://<SERVER_IP>/health` или актуальный health endpoint проекта
- `ollama list`
- `systemctl is-active ollama`
- `systemctl is-active nginx`
- `docker compose ps` или эквивалент
- проверка авторизации admin, если создавался
- проверка endpoint настроек или моделей, если в проекте есть UI выбора моделей

Ожидаемый финальный результат:
- проект полностью работает на сервере
- сайт доступен извне по IP
- nginx настроен
- ollama установлен и запущен
- все нужные модели скачаны
- backend реально видит ollama
- admin создан и проверен, если это было указано
- автозапуск после reboot настроен

Формат финального ответа:
- IP или URL сайта
- где лежит проект на сервере
- какие сервисы запущены
- какие модели Ollama скачаны
- данные admin, если создавались
- что именно было исправлено по ходу
- остаточные замечания, только если они реально есть
```
